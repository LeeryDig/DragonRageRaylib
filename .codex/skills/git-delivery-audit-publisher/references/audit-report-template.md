# Audit Report Template

Usar esta estrutura como baseline do `report.md`.

## Scope

- nome curto da entrega
- data e hora
- branch
- remote esperado

## Working Tree

- resumo de `git status --short`
- staged
- unstaged
- untracked

## Diff Summary

- diff stat unstaged
- diff stat staged
- principais arquivos impactados

## Validation

- testes rodados
- checagens feitas
- lacunas de validacao

## Publish Plan

- paths para `git add`
- mensagem de commit proposta
- remote/branch de push

## Outcome

- commit SHA
- push executado ou nao
- estado final do working tree
- riscos restantes
